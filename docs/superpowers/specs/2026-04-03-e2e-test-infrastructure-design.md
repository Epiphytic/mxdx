# E2E Test Infrastructure Design

## Goal

Unified end-to-end test infrastructure that gates PRs and releases, tests release binaries (Rust + npm/WASM), and produces performance reports for regression tracking.

## Architecture

A single shell script (`scripts/e2e-test-suite.sh`) orchestrates both Rust and npm E2E test suites against the beta infrastructure. Tests run against release-profile binaries by default, with debug as a fallback for troubleshooting. Test results are published as JUnit XML (for CI rendering) and performance JSON (for regression tracking). E2E tests gate both PR merges and releases.

## Components

### 1. Unified Test Script

**File:** `scripts/e2e-test-suite.sh`

**Interface:** `./scripts/e2e-test-suite.sh --profile release|debug [--skip-build]`

**Behavior:**

1. Unless `--skip-build` is passed: build Rust binaries with the specified profile (`cargo build --release` or `cargo build`) and build WASM (nodejs target) via `wasm-pack`
2. Set `MXDX_BIN_DIR` to `target/release/` or `target/debug/` (based on profile)
4. Set `TEST_PERF_OUTPUT` to `test-results/rust-e2e-perf.json`
5. Run Rust E2E tests: `cargo test -p mxdx-worker --test e2e_profile -- --ignored --test-threads=1`
6. Collect JUnit XML from Rust tests via `cargo2junit` (piped from `cargo test --format json`)
7. Set `TEST_PERF_OUTPUT` to `test-results/npm-e2e-perf.json`
8. Run npm E2E tests: `node --test --test-reporter=junit --test-reporter-destination=test-results/npm-e2e-junit.xml` against beta server tests
9. On success (both suites pass): run `node scripts/purge-test-accounts.mjs` to clean up devices and rooms
10. On failure: skip purge (preserve state for debugging), exit non-zero

The `--skip-build` flag allows CI jobs that build in a prior step to skip the redundant build. The script still sets `MXDX_BIN_DIR` based on profile.

**Default profile:** `release`

### 2. Rust E2E Test Harness Changes

**Binary resolution (`cargo_bin()`):**

The existing `cargo_bin()` function in `e2e_profile.rs` resolves binaries relative to the test executable in `target/debug/`. Add `MXDX_BIN_DIR` env var override:

```rust
fn cargo_bin(name: &str) -> PathBuf {
    if let Ok(dir) = std::env::var("MXDX_BIN_DIR") {
        let path = PathBuf::from(dir).join(name);
        assert!(path.exists(), "Binary not found at {}", path.display());
        return path;
    }
    // existing fallback
    let mut path = std::env::current_exe().expect("...");
    path.pop(); path.pop();
    path.push(name);
    assert!(path.exists(), "Binary not found at {}", path.display());
    path
}
```

**Performance JSON output:**

The existing `report()` function prints a table line to stderr. Extend it to also append a JSON entry to the file at `TEST_PERF_OUTPUT` (if set):

```json
{
  "name": "profile_echo_local",
  "transport": "mxdx-local",
  "duration_ms": 21400,
  "exit_code": 0,
  "stdout_lines": 46,
  "status": "pass"
}
```

The E2E script wraps individual entries with suite-level metadata:

```json
{
  "suite": "rust-e2e",
  "profile": "release",
  "timestamp": "2026-04-03T12:00:00Z",
  "git_sha": "abc123",
  "tests": [ ... ]
}
```

**JUnit XML:** Generated by piping `cargo test --format json` through `cargo2junit`. The E2E script handles this — no harness changes needed beyond `cargo test` format flags.

### 3. npm E2E Test Changes

**Test reporter:** Node 22 built-in JUnit reporter:

```
node --test \
  --test-reporter=spec --test-reporter-destination=stdout \
  --test-reporter=junit --test-reporter-destination=test-results/npm-e2e-junit.xml \
  packages/e2e-tests/tests/public-server.test.js \
  packages/e2e-tests/tests/public-session-persistence.test.js
```

**Performance JSON:** Tests that hit beta servers write timing entries to `TEST_PERF_OUTPUT` if set, using the same JSON schema as Rust tests.

**Scope:** Only tests requiring `test-credentials.toml` and beta infrastructure run in the E2E gate. Tests using local tuwunel instances remain in the existing `npm-e2e` CI job.

### 4. CI Pipeline (`ci.yml`)

**New `e2e-release` job:**

- Depends on `build` job
- Condition: `if: secrets.TEST_CREDENTIALS_TOML != ''` — skips for fork PRs (secrets unavailable)
- Steps:
  1. Checkout, install Rust 1.93.1, Node 22, wasm-pack, system deps, tmux
  2. Write `test-credentials.toml` from `TEST_CREDENTIALS_TOML` secret
  3. Run `./scripts/e2e-test-suite.sh --profile release`
  4. Upload `test-results/` as artifact (JUnit XML + perf JSON)
  5. Render JUnit XML via `dorny/test-reporter`
- Required status check for PR merge

**Fork PR behavior:** When `TEST_CREDENTIALS_TOML` is empty, the `e2e-release` job skips. The job reports a neutral/skipped status, not a failure. Unit tests still gate fork PRs.

### 5. Release Pipeline (`release.yml`)

**Split into two jobs:**

**Job 1: `e2e-gate`**
- Checkout, install toolchain
- Write `test-credentials.toml` from secret
- Run `./scripts/e2e-test-suite.sh --profile release`
- Upload test results as artifacts
- Build release binaries: `cargo build --release -p mxdx-worker -p mxdx-client`
- Upload release binaries as artifacts for the next job
- Render JUnit XML via `dorny/test-reporter`

**Job 2: `release`**
- Depends on `e2e-gate` (only runs if E2E passes)
- Downloads binary artifacts from `e2e-gate`
- Runs `npx semantic-release`
- Publishes:
  - Library crates to crates.io (existing order)
  - Binary crates (`mxdx-worker`, `mxdx-client`) to crates.io
  - npm packages (existing order)
  - Pre-built binaries as GitHub Release assets:
    - `mxdx-worker-x86_64-unknown-linux-gnu`
    - `mxdx-client-x86_64-unknown-linux-gnu`

### 6. Release Binary Publishing

**GitHub Release assets:** The `e2e-gate` job builds release binaries and uploads them as workflow artifacts. The `release` job downloads them and attaches to the GitHub Release created by semantic-release.

**crates.io binary crates:** Add `mxdx-worker` and `mxdx-client` to `scripts/publish-crates.sh`, published after all library crates. Requires proper `[package]` metadata (description, license, repository) in their `Cargo.toml`.

**Publish order:**
1. Library crates: mxdx-types → mxdx-matrix → mxdx-policy → mxdx-secrets → mxdx-launcher → mxdx-web → mxdx-core-wasm
2. Binary crates: mxdx-worker, mxdx-client
3. npm packages: @mxdx/core → @mxdx/launcher → @mxdx/client → @mxdx/web-console → @mxdx/cli

### 7. Test Report Artifacts

**JUnit XML:** Rendered by `dorny/test-reporter` as GitHub Check annotations. Failed tests show inline on PRs.

**Performance JSON:** Uploaded as workflow artifacts with 90-day retention. Contains git SHA and timestamp for self-describing data. Manual comparison for now — download two artifacts and diff. Future: automated `scripts/compare-perf.mjs old.json new.json`.

**Artifact structure:**
```
test-results/
  rust-e2e-junit.xml
  npm-e2e-junit.xml
  rust-e2e-perf.json
  npm-e2e-perf.json
```

### 8. Account Hygiene

**On E2E success:** `node scripts/purge-test-accounts.mjs` runs automatically. Leaves all rooms, logs out all sessions (deletes all devices), cleans local crypto stores. Both servers (ca1-beta, ca2-beta) are purged.

**On E2E failure:** Purge is skipped. Devices, rooms, and crypto state remain for debugging. Next successful run cleans up.

**Rationale:** CI runners have no persistent keychain or crypto store. Every run creates fresh Matrix devices. Without purging, accounts accumulate stale devices and rooms that interfere with subsequent runs (room disambiguation failures, key exchange timeouts).

### 9. Secret Management

**Single secret:** `TEST_CREDENTIALS_TOML` contains the full `test-credentials.toml` file contents. Written to disk at job start:

```yaml
- name: Write test credentials
  if: env.TEST_CREDENTIALS_TOML != ''
  run: echo "$TEST_CREDENTIALS_TOML" > test-credentials.toml
  env:
    TEST_CREDENTIALS_TOML: ${{ secrets.TEST_CREDENTIALS_TOML }}
```

**Contents:** Both beta servers, both test accounts, coordinator account. Same format as the local `test-credentials.toml`.

**Existing individual secrets** (`TEST_SERVER_URL`, `TEST_ACCOUNT*`) in the `npm-public-server` job remain for backward compatibility but are not used by the new E2E infrastructure.

## Performance JSON Schema

```json
{
  "suite": "rust-e2e | npm-e2e",
  "profile": "release | debug",
  "timestamp": "ISO-8601",
  "git_sha": "string",
  "tests": [
    {
      "name": "string",
      "transport": "mxdx-local | mxdx-federated | ssh",
      "duration_ms": 0,
      "exit_code": 0,
      "stdout_lines": 0,
      "status": "pass | fail | skip"
    }
  ]
}
```
