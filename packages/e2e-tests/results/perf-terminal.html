Interpretation

  • P2P provides near-instantaneous terminal I/O suitable for interactive shell sessions
  • Matrix (Tuwunel) adds E2EE + sync overhead but remains usable for command execution
  • Matrix (public, matrix.org) is rate-limited to ~1 event/sec, making it unsuitable for interactive terminals without P2P
  • The BatchedSender adapts: 5ms batching for P2P (keystroke-level), 200ms for Matrix (rate-limit safe)
  • AES-256-GCM encryption overhead is negligible (<1ms per encrypt+decrypt)
